# PART A ‚Äî WRITE A TUTORIAL (Markdown)

From the input document:

1. Extract and organize:

   * Key concepts
   * Definitions
   * Rules / patterns / conditions
   * All important formulas / equations and any worked-example patterns

2. For EACH major formula or definition:

   * First, state it clearly and correctly.
   * Then **explain its meaning in simple language**:

     * What does it represent in real terms?
     * How should a Grade 11‚Äì12 student interpret it?
     * Units: what are they?
     * What does each symbol mean?
   * If relevant, include **one short worked example or interpretation** showing how it is used.

3. Structure the tutorial as:

   * Section headings that match or closely follow the document sections (e.g., 7.1, 7.2, etc.).
   * Under each section, include:

     * **Key ideas**
     * **Important formulas & their meaning**
     * **Short example(s)** (if available)
   * Do **NOT** include original page numbers.

4. Keep explanations concept-focused:

   * Explain **WHY**, not just HOW.
   * Be clear, compact, student-friendly. Avoid overly decorative writing.

# PART B ‚Äî BUILD A SWITCHABLE MULTI-STYLE HTML LEARNING VIEW (Single Self-Contained HTML)

Create **one** self-contained HTML file that presents the same content from Part A in **three switchable styles**, and supports **two switchable themes** (**Daylight default** + **Nightlight**).

## A) Style Modes (must be switchable)

Add a top toolbar with a **Mode Switch** (tabs or segmented buttons) that toggles between:

### Mode A ‚Äî ‚ÄúBook View‚Äù (simple textbook)

* Clean reading layout (like a mini e-textbook).
* Section headings (7.1, 7.2‚Ä¶) with full readable paragraphs.
* Optional callout boxes for ‚ÄúKey idea‚Äù, ‚ÄúPitfall‚Äù, ‚ÄúTip‚Äù.

### Mode B ‚Äî ‚ÄúPPT View‚Äù (slide-like)

  * Slide 1: Big idea / overview
  * Slide 2: Definitions (key terms)
  * Slide 3: Formulas + symbol meanings
  * Slide 4: Example(s) / interpretation
* Provide **Next/Prev** buttons and keyboard support (‚Üê ‚Üí).
* Use large fonts and clear contrast (**works well in both themes**, with Daylight as default).

### Mode C ‚Äî ‚ÄúInteractive Map View‚Äù (interactive style)

## B) Global UI/UX Requirements (apply to all modes)

### 1) Themes: Daylight (default) + Nightlight (switchable)

Add a **Theme Toggle** in the top toolbar (button or segmented control) to switch between:

#### Theme 1 ‚Äî Daylight (DEFAULT)

* Bright, normal daylight UI:

  * Background: white / very light gray
  * Text: near-black / dark gray
  * Borders: light gray
  * Cards: white with subtle shadow
* High readability (good for printing and daytime viewing).

#### Theme 2 ‚Äî Nightlight (OPTIONAL MODE)

* A **warm, eye-comfort night reading** theme (NOT pure black):

  * Background: deep warm gray / very dark brown-gray (not #000)
  * Text: soft off-white / warm light gray (not pure white)
  * Borders: muted dark gray
  * Cards: slightly lighter than background, subtle shadow
* Keep contrast readable, reduce glare, avoid neon colors.

#### Implementation requirements

* Use CSS variables for theme colors (e.g., `--bg`, `--text`, `--card`, `--border`, `--shadow`).
* Toggle theme by applying a state/class to the root (examples conceptually):

  * `body[data-theme="day"]` (default)
  * `body[data-theme="night"]`
* Remember the user choice using `localStorage` so theme persists on refresh.
* Default must be **Daylight** if no stored preference exists.

### 2) Layout + Sidebar Behavior (CRITICAL)

* Include a left sidebar/nav panel with:

  * Section links (scroll-to)
  * A search/filter input that filters sections/cards by keyword (works across all modes)

* The sidebar/nav panel must be **show/hide** via a button in the top toolbar.

* **When the sidebar is hidden:**

  * The main content area must **expand to occupy 100% of the available screen width**
  * No ‚Äúempty left space‚Äù should remain
  * Implement this via a layout state/class
    (example conceptually: `body.sidebar-hidden` changes grid/flex so content becomes full-width)

* **Mobile-friendly**:

  * On narrow screens, the sidebar moves **above** the content (top stacked layout)
  * Sidebar show/hide still works on mobile

### 3) Interactions (all modes)

* Collapsible nodes (click header to expand/collapse) where relevant
* Small **copy buttons üìã** beside key formulas/snippets to copy to clipboard
* **Double-click** on any text block copies that block to clipboard
* No external CSS/JS: everything inline in one HTML file.
